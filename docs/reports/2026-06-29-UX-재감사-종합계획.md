## 0. 한눈 요약

- **데이터 규모**: 6개 렌즈 83건 → 중복 제거 58건 → 코드 대조 검증(이미 해소 0건, 전부 실재). + IA/내비 재검토 11건(치명 2 신규) + 누락 비평 12종. 디렉터가 핵심 항목(터치타겟·가입동선·토큰체계·스크롤복원·디렉터리진입·사진첨부)을 직접 코드로 재확인.
- **기존 문서 대비**: 내비/검색/리포트 구조 개편은 끝났으나, 옛 문서의 [치명]·[개선] **대부분은 그대로 유효**합니다(터치타겟·토큰 이중체계·끄적끄적·하트색·CTA/톤 혼용 등). 즉 "구조"는 바뀌었지만 "디테일·접근성·일관성" 부채는 미해결입니다.
- **가장 무거운 신규 발견 5가지**:
  1. (치명) **핵심 가치 진입점 부재** — 전문의/원장 디렉터리(`/doctors`)가 5탭·헤더 어디에도 없음(푸터·about·404만).
  2. (치명) **셸 스크롤 복원 깨짐** — `ScrollManager`는 `window` 기준인데 실제 스크롤러는 `.root`. 뒤로가기 위치 복원 무력화(런타임 확인 필요).
  3. (치명) **디자인 토큰 이중 체계** — `app.module.css(.root)`가 `globals.css`와 별개 토큰 세트를 자체 정의(값까지 갈림), 291회 사용.
  4. (치명) **카드 액션 터치타겟 22px** — 좋아요·댓글·저장·공유가 패딩 없이 22px, WCAG/HIG 미달.
  5. (개선·우선) **가입 동선 혼란 루프** — "회원가입" 링크가 게스트를 `/login`으로 되돌림. 이메일 폼은 로그인 전용(가입 불가)인데 가입되는 듯 보임. (가입 자체는 소셜로 가능 → 완전차단 아님)

## 2. [개선] — 전환·일관성·접근성·심미

### 2.1 정보구조(IA) — 신규
- `personal-data-triple-overlap` — 투데이/내노트/마이가 개인 데이터(노트·후기·활동 카운트)를 **세 화면에서 다른 라벨·다른 KPI 정의**로 중복. 같은 "내 노트" 링크·"내가 쓴 글" 카운트 중복.
- `settings-redirect-to-profile` — `/settings`·`/settings/profile`이 모두 공개 프로필 `/{handle}`로 redirect. 마이의 "앱 설정/내 피부 정보/탈퇴/프로필"이 전부 같은 곳으로 수렴(멘탈모델 위반). 코드 주석도 "전용 페이지 없음" 명시.
- `write-entry-inconsistent` — 글쓰기 진입 4종(FAB/헤더/사이드CTA/투데이 히어로) 도착지·노출규칙 불일치. 프로필·토픽에서는 FAB가 사라져 모바일 글쓰기 진입 불가.
- `today-vs-feed-ambiguous` — 투데이와 피드 역할 경계 모호. "관심 키워드 새 글 전체보기 →`/`"가 키워드 컨텍스트를 잃음(→ `/?q={kw}`로 보내야 정합).
- `shop-tab-occupies-slot` — 쇼핑 탭이 5탭 중 1슬롯 영구 점유(준비중). 가장 비싼 하단탭 자원 낭비, 매 클릭 토스트=학습된 좌절.

### 2.2 온보딩·전환
- `onboarding-gate-too-many` — 첫 가입에 얼굴형·피부타입·피부고민·관심키워드까지 전부 필수(InterestPicker 무거움) → 마지막 이탈 결정타.
- `onboarding-sequential-validation` — 검증이 한 필드씩 순차 + 에러가 화면 맨 아래 고립(미충족 섹션은 위쪽).
- `onboarding-email-redundant` — OAuth로 받은 이메일을 다시 편집 가능 입력으로 노출(중복 마찰).
- `write-fab-guest-gate` — 게스트가 글쓰기 누르면 빈 게이트 카드 벽(작성 의도 최고 순간). next 보존은 되나 모달이 더 부드러움.
- `engagement-threshold-high` — 가입권유 임계 15점·dismiss 7일 침묵. **단 오너가 v3에서 6→15 의도적 상향**(주석 근거 명시) → 충돌, 측정 의지 있을 때만.
- `engagement-trust-copy-random` — 가입권유 트러스트 카피 매 노출 랜덤(측정 불가).
- `push-toggle` — 푸시 토글 제목 이모지(🔔/✓) + 가치 시점 in-context 권유 없음.

### 2.3 심미·디자인 토큰(토큰 통합이 공통 선행)
- `brand-blue-4-variants` — 브랜드 블루 4종(#4CBFF2/#45B7E8/#2A9FD6/#1B87C9) 혼재 + 흰배경 텍스트가 #4CBFF2(~2.4:1)로 AA 미달(ReviewSummary·CardBody·리포트 히어로). `--primary-active`(#1B87C9) 도입됐으나 텍스트에 미적용.
- `card-radius-16-vs-24`(부분) — 카드 모서리 24px(.card) vs 16px(신규 리포트). globals에 24px 슬롯 없음. (단 "같은 화면 혼용" 서술은 정정—화면 간 차이임)

- `category-fill-button-contrast`(제안) — 오렌지/올리브/블루그레이 채움 버튼 흰글씨 AA 미달. **메모리 결정(`feedback-primary-cta-skyblue`: 채움 버튼 글자는 항상 흰색)과 충돌** → 재확인 필요.
- `chip/리포트 색·모서리 리터럴` — 칩·리포트 보조색(#E2E7EC·#64748B·#8A9099·#E6F2FA 등)·999px·비표준(10/14/18/22px) 리터럴 산발. 토큰 통합에 흡수.
- `font-stack-variable-mismatch` — `--font`가 'Pretendard Variable' 1순위인데 변수폰트 미로드(정적만). 죽은 1순위 + 스택 dual 정의.
- `dark-mode-unsupported`(후속) — prefers-color-scheme 0건, 라이트 고정. 토큰 통합 후 최소 다크 세트.
- `heart-overlay-color-mismatch` — 더블탭 하트(하늘색 --primary) ≠ 좋아요 활성(코랄 --accent).

### 2.4 인터랙션·접근성
- `comment-meta-buttons-tiny-target` — 댓글 답글·좋아요·메뉴(⋮) 11~16px 타겟.

### 2.5 카피·톤(해요체) — 대부분 문자열 한 줄 교체, 일괄 처리 권장

## 3. 누락 비평 — 아무도 다루지 않은 사각지대(코드 확인됨)

1. **(큰 공백) 회원 글 본문 사진 첨부 자체가 없음** — 카드 에디터는 외부 링크(YouTube)+PubMed만. 단 `/api/upload`(EXIF 제거·sharp·8MB)는 **구축돼 있으나 프런트 미연결**(아바타는 storage 직접). → 추가 비교적 수월(스코프 결정).
2. **모달 Tab 포커스 트랩 부재**(6종 전수) — aria-modal·초기포커스는 있으나 Tab이 배경으로 탈출.
3. **하단탭 `aria-current="page"` 부재** — SR이 현재 탭 모름.
4. **토스트가 화면 중앙 단발·1.5초·큐 없음** — 연속 동작 시 덮어쓰기, 시선 이탈.
5. **빈/오류 상태가 제3의 레거시 토큰(--tt-*/--ink-*)** — FeedView 빈결과 칩.
6. **알림 '내 기록' 탭이 영구 하드코딩 더미 데이터**(RECORD_NOTIS 3건, '예시' 배지).
7. **오프라인 배너가 텍스트 한 줄**·복구 동선 없음·navigator.onLine만 신뢰.
8. **폼 인라인 유효성 피드백 부재**(글쓰기·프로필 편집 전반).
9. **검색 자동완성이 부분일치 only**(초성/오타/순위 없음, procedure-dict 미연결).
10. **이미지 alt 일괄 빈 문자열**(아바타 등 정보성 이미지도 장식 처리).
11. **쇼핑 탭 영구 준비중**(§2.1과 동일).
12. **한글 IME 조합 중 Enter 처리** 검색·댓글 점검 필요(런타임).
13. **데드 컴포넌트 `today/RecordTab.tsx`** — 미사용·동선이 라이브와 반대(IA 혼선).
14. **토픽(`/topics`) 분류 내비 고아** — 검색(`/?q=`)으로만 도달, 브라우징 동선 없음.

## 5. 실행 묶음 제안

토큰 통합이 다수 심미 항목의 **공통 선행**이라 회귀 위험이 크므로 한 덩어리로 신중히. 빠른 접근성·카피는 먼저 체감 확보.

- **묶음 A — 빠른 픽스(저위험·단일파일)**: 카드/댓글/칩 터치타겟, skip-link·aria-current·aria-expanded·모달 포커스트랩, 하트색 1줄, 더블탭 onDoubleClick, 로그인 placeholder.
- **묶음 B — 톤·카피 일괄**: §2.5 어체/형태/표현/이모지 전부(문자열 위주). '끄적끄적'·'투데이'는 §6 결정 후.

## 8. 부록 — 쉬운 풀이(자연어 설명)

기술 용어가 많아 어려운 부분을, 비전문가도 이해할 수 있게 비유 중심으로 풀어 적습니다. 괄호 안 ID로 위 본문 항목과 연결됩니다.

### 8.1 자주 나오는 용어 한 줄 풀이

- **디자인 토큰 / SSOT(단일 출처) / 이중 체계** — 색·글자크기·모서리 둥글기 같은 디자인 값을 매번 손으로 적지 않고 "`--primary` = 하늘색"처럼 **이름표 하나에 정의해 두고 모든 화면이 그 이름표를 참조**하는 방식이 '디자인 토큰'입니다. SSOT(Single Source Of Truth)는 "그 이름표 정의가 딱 한 곳에만 있어야 한다"는 원칙. 정의가 두 곳에 흩어진 게 '이중 체계'이고, 한쪽만 고치면 화면마다 색이 달라집니다.
- **WCAG / 터치타겟 44px·24px / HIG** — WCAG는 국제 웹 접근성 표준, HIG는 애플의 디자인 지침. '터치타겟'은 손가락으로 누를 수 있는 버튼의 실제 크기입니다. 손끝 평균이 약 9~10mm라 애플은 44px(약 1cm), 구글은 48px, 최소 기준(WCAG 2.5.8)은 24px을 권합니다. 이보다 작으면 옆 버튼을 잘못 누릅니다(오탭).
- **스크린리더 / ARIA** — 시각장애인이 화면을 '귀로 듣는' 보조기기가 스크린리더입니다. ARIA는 버튼·탭의 상태(눌림/펼침/현재 위치)를 스크린리더에게 **말로 알려주는 표지판**. 표지판이 없으면 "지금 좋아요가 눌린 상태인지" 같은 걸 못 읽어줍니다.
  - `aria-pressed`=눌림 여부, `aria-expanded`=펼쳐짐 여부, `aria-current`=지금 보고 있는 탭, `skip-link`=반복되는 메뉴를 건너뛰고 본문으로 바로 가는 숨은 링크, **포커스 트랩**=팝업이 떠 있을 때 키보드 이동이 팝업 밖으로 새지 않게 가두는 것, `role=tab`="이걸 탭이라고 선언했으면 ←/→ 화살표 키로도 넘어가야 한다"는 규약(반쪽만 지키면 거짓 표지판).
- **라우팅 / 딥링크 / OAuth / 세션** — 라우팅=주소(URL)에 따라 어떤 화면을 보여줄지 결정. 딥링크=특정 화면으로 바로 가는 주소. OAuth=구글·카카오 계정으로 '비밀번호 없이' 로그인하는 방식. 세션=로그인된 상태(쿠키로 유지).
- **스크롤 컨테이너(window vs .root) / 스크롤 복원** — 보통 웹은 '브라우저 창 전체(window)'가 스크롤되는데, 이 앱은 화면을 꽉 채운 상자(`.root`)를 따로 두고 **그 상자 안만 스크롤**합니다. '스크롤 복원'은 글을 읽다 다른 글로 갔다 뒤로가기 하면 **읽던 위치로 되돌려주는** 기능인데, 시스템이 '창(window)' 위치만 기억하고 실제로는 '상자(.root)'가 움직이니 엉뚱한 0(맨 위)을 기억해 복원이 안 됩니다.
- **그 외** — `sticky`=스크롤해도 화면에 붙어 따라오는 고정. `마이크로인터랙션`=버튼 누를 때 살짝 튕기는 작은 애니메이션(만족감). `masonry`=핀터레스트식 벽돌 쌓기 레이아웃. `EXIF`=사진에 박히는 촬영 위치·시각 정보(개인정보라 제거 필요). `다크모드(prefers-color-scheme)`=OS가 야간 모드면 앱도 어둡게. `Variable 폰트`=굵기를 무단계로 조절하는 폰트(반대는 정해진 굵기 몇 개만 있는 '정적 폰트').

### 8.2 5대 치명 항목 — 쉬운 설명

각 항목을 "무슨 일이 일어나는가 / 비유 / 사용자가 겪는 것" 순으로 풉니다.

1. **전문의·원장 디렉터리 진입점 부재 (`doctors-entry-missing`)**
   앱은 "피부과 전문의가 직접 답해준다"를 간판으로 내세웁니다. 그런데 "어떤 원장님들이 있나" 보러 가는 메뉴 버튼이 하단 5개 탭에도 상단에도 없습니다. 그 페이지(`/doctors`)는 만들어져 있는데, 맨 아래 푸터(저작권 줄)·'소개' 페이지·404 오류 페이지에서만 링크가 걸려 있어요.
   - 비유: 식당 간판은 "유명 셰프의 요리"인데, 정작 셰프 소개 코너로 가는 안내판이 매장 어디에도 없고 영수증 뒷면에만 적힌 격.
   - 사용자: 신뢰의 핵심(전문의)을 보고 싶어도 길을 못 찾습니다.

2. **셸 스크롤 복원 깨짐 (`shell-scroll-restore-broken`)**
   위 용어 풀이대로, 앱이 '상자(.root)' 안에서 스크롤되는데 위치 기억 장치는 '창(window)'만 봅니다. 그래서 피드를 한참 내려 보다 글 하나를 눌러 들어갔다 **뒤로가기 하면 맨 위로 튕겨 올라갑니다**(피드는 따로 보정한 흔적이 있어 부분적으로만 동작).
   - 비유: 책에 책갈피를 꽂았는데 그 책갈피가 다른 책에 꽂혀 있어, 돌아오면 처음부터 다시 펴야 하는 상황.
   - 단, 어느 화면에서 얼마나 튕기는지는 브라우저로 직접 눌러봐야 확정됩니다(그래서 '런타임 확인 필요'로 표시).

3. **디자인 토큰 이중 체계 (`design-token-dual-system`)**
   색·모서리 같은 디자인 '이름표'가 두 군데에 따로 정의돼 있고 심지어 **값이 서로 다릅니다**(같은 '브랜드 하늘색'인데 한쪽 #4CBFF2 / 다른 쪽 #45b7e8, 카드 모서리도 한쪽 12px / 다른 쪽 24px). 게다가 앱 본문 쪽 파일은 주석에 "이건 globals와 따로 논다"라고 대놓고 적혀 있어요(원래 '미리보기 스킨'으로 만들었다 그대로 본 앱이 된 흔적).
   - 비유: 같은 회사 로고 색을 디자인팀과 인쇄팀이 각자 다른 잉크로 정의해 둬서, 명함과 간판 색이 미묘하게 다른 상황.
   - 사용자: 한 화면 안에서, 또 화면을 옮길 때 파랑·모서리가 미묘하게 달라 '정돈 안 된' 느낌. 개발 입장에선 색 하나 바꾸려면 두 곳을 따로 고쳐야 합니다.

4. **카드 액션 터치타겟 22px (`card-actions-touch-target-22px`)**
   피드 카드 아래 좋아요·댓글·저장·공유 아이콘이 22px(약 5.8mm)인데 버튼에 여백(패딩)이 없어 누를 수 있는 영역이 딱 아이콘 크기뿐입니다. 손끝(약 1cm)보다 작아요.
   - 비유: 엘리베이터 버튼이 콩알만 해서 옆 층을 자꾸 누르는 격.
   - 사용자: 모바일에서 좋아요 누르려다 댓글이 열리는 등 오탭. 고치는 법은 아이콘 크기는 두고 **버튼 둘레에 보이지 않는 여백만 키우면** 됩니다(시각은 그대로, 누를 영역만 확대).

5. **가입 동선 혼란 루프 (`login-signup-deadend`)**
   로그인 화면은 ①위쪽 "구글·카카오로 시작" ②"또는 이메일로 로그인" ③맨 아래 "처음이신가요? 회원가입"으로 돼 있습니다. 실제 가입은 ①소셜로만 됩니다. 그런데 ③"회원가입"을 누르면 `/signup`으로 갔다가, 로그인 안 한 사람이면 다시 로그인 화면으로 되돌아옵니다(그 페이지는 원래 소셜 로그인 직후 거치는 '가입 마무리'용이라). 또 ②이메일 폼은 '가입'은 안 되고 '로그인'만 되는데 마치 가입도 될 것처럼 보입니다.
   - 비유: '신규 가입' 문을 열고 들어갔더니 다시 입구로 나와 버리는 회전문.
   - 사용자: 가입 자체는 소셜로 되니 완전히 막힌 건 아니라 [치명]→[개선]으로 내렸지만, 이메일로 가입하려는 사람은 헛걸음하고 신뢰가 깎입니다.

### 8.3 헷갈리기 쉬운 개선 항목 풀이

- **개인데이터 삼중 중복 (`personal-data-triple-overlap`)** — 투데이·내노트·마이 세 화면이 "내가 쓴 글 수·내 노트·내 후기" 같은 같은 정보를 **서로 다른 묶음, 다른 숫자 기준으로** 보여줍니다. 사용자는 "내 후기 어디서 보지?"를 매번 헷갈려요. → 역할을 나눠야 합니다(투데이=오늘 위젯, 내노트=내 기록 단일 장소, 마이=계정·설정).
- **설정이 공개 프로필로 수렴 (`settings-redirect-to-profile`)** — 마이의 "앱 설정·내 피부 정보·탈퇴·프로필"을 누르면 **전부 같은 내 공개 프로필 페이지로** 갑니다(설정 전용 페이지가 없어서). "앱 설정"을 눌렀는데 내 프로필이 뜨면 기대와 어긋납니다.
- **토픽 내비 고아 (`/topics`)** — 시술을 분류해 모아 보는 '토픽' 페이지가 있는데, 앱 안에서는 검색으로만 닿고 "전체 분류를 둘러보는" 메뉴가 없어 사실상 외톨이입니다(검색으로 들어오는 외부 방문용으로만 살아 있음).
- **사진 첨부 부재 + 업로드 백엔드는 존재 (사각지대 1)** — 회원이 글에 사진을 못 붙입니다(에디터가 유튜브 링크·논문 참고문헌만 받음). 그런데 사진을 안전하게 받는 서버 기능(`/api/upload`, 위치정보 EXIF 제거까지)은 **이미 만들어져 있는데 작성 화면에 선만 안 이어진** 상태입니다. 즉 마음먹으면 비교적 쉽게 붙일 수 있습니다(다만 '사진 기능을 넣을지'는 제품 방향 결정).
- **브랜드 블루 4종 + 흰배경 글자 대비 (`brand-blue-4-variants`)** — 같은 '브랜드 하늘색' 역할이 코드에 4가지 파랑으로 떠다니고, 흰 바탕 위 글자에 가장 연한 파랑(#4CBFF2)을 써서 **글자가 흐릿하게** 보입니다(명암 대비 약 2.4:1, 기준 4.5:1 미달). 이미 진한 파랑(#1B87C9)이 있는데 글자엔 안 쓰고 있어요.
- **모달 포커스 트랩 부재 (사각지대 2)** — 팝업(확인창 등)이 떴을 때 키보드 Tab을 누르면 포커스가 팝업 뒤 배경으로 빠져나갑니다. 키보드·스크린리더 사용자는 팝업 안에 머물러야 하는데 새어나갑니다.
- **검색 자동완성이 부분일치만 (사각지대 9)** — 검색어 추천이 글자가 똑같이 들어간 것만 찾습니다. "ㅂㅌ"(초성)이나 "보톡쓰"(오타)로 치면 결과가 0이 되고, 인기·최신 순위 없이 사전 순서대로만 나옵니다(이미 있는 시술명 교정 사전과 연결돼 있지 않음).
- **알림 '내 기록' 탭 더미데이터 (사각지대 6)** — 알림함의 두 탭 중 하나가 실제 데이터가 아니라 항상 똑같은 예시 3건("스킨부스터 권장 주기가 다가왔어요" 등)을 보여줍니다('예시' 배지는 있음).
