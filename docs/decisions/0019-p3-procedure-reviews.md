## Context

피부텐텐에 "시술 후기"를 도입한다. 단, 의료광고법(의료법 §56② 치료경험담·환자후기 규제)과 충돌 가능성이 있고, 상위·하위 시술 계층 집계가 현 시스템에 전혀 없다(태그검색은 정확일치 + 의사 qa글 한정). 따라서 데이터 모델·집계·노출 정책을 먼저 확정한다.

후기 대상 시술은 원장님 선별로 정식 31종(리프팅 17 + 주입 14) + 하위 14종으로 확정(`시술태그_선별표.md`).

## Decision

### 1. 카드 4종
- `post`(끄적끄적) · `qa`(의사 Q&A) — 기존.
- `review`(개별 시술후기 글) · `review_summary`(시술 리포트 = 시술별 자동 집계 카드) — 신설.

### 2. 분류 체계 — `procedure_taxonomy` (P3-a, 마이그 0199)
- 2계층. `parent_ko IS NULL` = 정식 시술(31), 그 외 = 하위(상위 ko 참조).
- 하위 14: 보톡스←(나보타·메디톡신·제오민·코어톡스·휴톡스·리즈톡스), 쥬베룩←(볼륨·아이), 리쥬란←(아이·HB), 덴서티←(알파팁), 세르프←(아이), 벨로테로←(리바이브), 레스틸렌←(비탈라이트).
- en slug 은 `procedure-mappings.json` 매핑 재사용(누락 0).

### 3. 작성 주체·검수
- 시술후기는 **일반 회원**이 작성.
- 검수에서 **병원명·의사명 노출을 차단**(신규 규칙) + 기존 의료광고 금지표현 필터 재사용. 환자 직접후기의 의료광고성 리스크를 표현 단위로 통제.

### 4. 정량 입력
- **필수 5**: 만족도 · 효과 체감 · 통증 · 회복기간 · 추천 의향. 시술명도 필수.
- **선택**: 시술 부위 · 비용 만족도 · 자유 후기 본문.
- 효과 체감 분야(동안/노안, 피부장벽/민감성)는 후기 속성으로 별도 관리(P3-b 스키마에서 확정).

### 5. 이중집계
- 하위 시술도 독립적으로 후기를 받음.
- 하위(예: 쥬베룩볼륨) 후기 → ① 하위 자체 통계 + ② 상위(쥬베룩) 통계에 **동시** 누적.
- 상위 통계 = 상위 직접 + 모든 하위 합산.
- **보톡스 6브랜드도 전부 이중집계**(브랜드별 자체 통계 유지). '통합' 구분 폐지.
- 검색: 상위 검색 → 상위+하위 노출, 하위 검색 → 하위+상위 노출.

### 6. 노출·SEO
- 개별 후기 카드(`review`): 피드 노출 O, 검색엔진 **noindex**.
- 시술 리포트(`review_summary`): **index**.

## Consequences

- `procedure_taxonomy` 를 RPC 가 JOIN 하여 이중집계·검색확장 구현(P3-d/e). JSON 사전 대신 DB 테이블 채택 — 서버측 집계가 깔끔.
- 검수에 병원·의사명 탐지 규칙 추가 필요(P3-b).
- 의료법 최종 적합성은 별도 법률 자문 권장(본 ADR 은 제품·기술 결정).
- 후속 단계: P3-b(procedure_reviews + API + review 카드 + 검수) → P3-c(입력 폼) → P3-d(리포트 집계) → P3-e(검색 확장·노출).
