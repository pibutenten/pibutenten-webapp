## 2. 현재 구조 핵심 (탐색 결과 요약)

### 2.1. 리포트 — "서버(SEO·집계) ↔ View" 분리가 깨끗함

- `app/reports/layout.tsx`(공유 셸, force-dynamic) + `ReportsShell`(AppShell + 사이드바 + 카테고리 컨텍스트 + PTR) 아래 허브(`ReportsIndexView`+`ReportsIndexCard`)와 상세(`ReportsDetailView`+`ReportsReviewCard`)가 있음. 메타·JSON-LD 는 각 `page.tsx` 소유 — **View 만 갈아끼우면 SEO 무손상** (ADR 0024/0025).
- 접힘 요약부 `ReportSummaryBox` 소비처는 **3곳**: ①`ReportsIndexCard`(접힘 헤더) ②`/topics/[tag]/TopicTagView`(닫힌 글상자) ③데이터 조립 측 `/topics/[tag]/page.tsx`(`getReportSummaryForTag` + 헤드라인 엔진 서버 계산) — 개편 시 3곳 동시 정합 확인 (D1).
- 허브 정렬은 클라 5종(최신/후기많은/재시술의향/만족도/통증적은), 기본 최신순. 무한스크롤 8개/페이지, 뒤로가기 sessionStorage 스냅샷 복원.
- 데이터: 신디자인 요구 12항목 중 **미수집 항목 없음**. 소규모 신규는 ①후기 카드 댓글 수 집계 ②리포트 진짜 저장 재배선 뿐 (양쪽 모두 마이그 불필요).
- 주의: `sat_dist` 배열 순서가 RPC 별로 다름 (pool=[1점..5점] 오름차순 vs admin 0328=[5점..1점]) — 혼용 금지.
- anon 은 `procedure_reviews` 24컬럼 중 21컬럼만 SELECT 가능 (`solo_price`·`visited_on`·`reactions` 제외) — 이번 시안엔 해당 노출 없음 → 작업 불필요.

### 2.2. 마이·프로필 — 목록·설정·전환·탈퇴가 전부 `/[handle]` 에 위임된 구조

- `/my` = `MyPageView` (인라인 스타일 위주 — 사실상 전면 재작성 대상). 카운트 5종 + PII 3종(연령대/얼굴형/피부타입, `get_profile_pii` RPC) 서버 집계 완비.
- `/[handle]` = `ProfileView` 6탭(posts/reviews/comments/likes/saves/skin) + isOwner 아코디언(`ProfileEditClient` 1,093줄 임베드 + `ClinicLinksSection` + `AccountSwitcherCard`) + 타인 `FollowButton`.
- 최근 본 글: DB 기반(`card_views` + `get_my_recent_views` RPC 0287) — `/my/recent` 전용 라우트 존재.
- **PII 는 `get_profile_pii` RPC 경유 필수** (마이그 0335 REVOKE — 직접 SELECT 시 42501 로 쿼리 전체 실패).
- **active 명함 RLS 경로 (검증 완료)**: `src/lib/supabase/server.ts` L44-46 이 identity 쿠키의 UUID 를 `x-active-profile-id` 헤더로 매 요청 주입 → DB `current_active_profile_id()` → RLS 가 active 명함 단위 동작. 서버 컴포넌트의 diaries 직접 조회도 이 경로로 정상 (현행 `/notes` 와 동일).
- 알려진 소결함: `/my/recent` 에 clinic 역할 redirect 누락 (admin/doctor 는 있음 — 실파일 L40-41 확인). Phase 3 에서 수정.

### 2.3. 디자인 시스템 — 신디자인 색 5종은 전부 신규

- 토큰 이원화: `globals.css`(:root, 리포트 계열이 소비) ↔ `app.module.css`(.root 스코프, 셸·스킨 페이지가 소비). 서로 비의존 — app.module.css 헤더 주석에 "globals 비의존 자체 정의" 격리 원칙 명문화.
- 신디자인 `#029688 / #1A9DE8 / #F5FBFF / #DAF1FB / #EAF2F8` — **코드베이스에 0건** (전부 신규 도입). 특히 `#029688` 은 기존 카테고리 초록 `#009688`(contour) 과 한 글자 차이의 **별개 값** — 혼동 주의.
- `#009688` 계열 하드코딩 사본 위치 (D2 교체 시 전수 동기 대상): `.catContour`(L1732 rgba(0,150,136,.15)/#1a756c) · `.tagBtnActive.catContour`(L1776 #1a756c) · **`button.t[data-cat="contour"]:hover`(L1413 rgba(0,150,136,0.15))** — 실행 시 `rgba(0,150,136` · `#1a756c` grep 전수 확인.
- 페이지별 배경 관행 없음. sticky 정렬 칩·헤더·상태바 필러가 `--tt-canvas`/`--tt-canvas-top` 을 참조하므로, **페이지 variant 클래스가 이 CSS 변수를 재정의**하면 참조처가 자동 추종된다 (§5 Phase 0).
- 아이콘 라이브러리 없음 — 파일 로컬 인라인 SVG 관행. 신규 세트는 공용 모듈 신설.
