# 병원 운영 프로그램 — 환자 관리 · 시술기록 관리 (최종 설계서, 260706)

> 상태: **최종 — 원장 확정 대기**. 4인 브레인스토밍(DB·목록·상세·IA) + 디자인 서브에이전트(동의화면) + 원장 2차 피드백 전량 반영.
> 목표: `/clinic` 을 **관리자 대시보드(/admin)와 동일한 UI/UX**로, 병원이 **① 환자 관리** 와 **② 시술기록 관리** 를 체계적으로 운영하는 프로그램.
> 상위 토대: `docs/plans/260704 병원계정 시술기록 대행입력 계획.md`(대행입력 RPC 0345). 본 문서는 운영 프로그램 고도화.

## 0. 확정 사항 (원장 지시)

| # | 결정 |
|---|---|
| C1 | **두 모듈 전부 구축** — 환자 관리 + 시술기록 관리(환자별 + 지점 전체 대장) |
| C2 | **연결 해제(revoked) 시 병원은 수정·작성 불가, 조회만** |
| C3 | **환자는 여러 병원 동시 연결 가능**(비귀속 다대다) |
| C4 | 시술기록 소유 = **회원**. **회원은 자기 시술노트(병원 대행분 포함)를 전부 수정·삭제 가능** |
| C5 | 회원 후기(procedure_reviews) 달린 기록은 병원이 삭제 불가(고아 방지) |
| C6 | **두 모듈 모두 그 환자의 시술기록으로 진입 가능** |
| C7 | **시술기록 관리 = 날짜별**. 목록 뷰 + **캘린더 뷰**(날짜 골라 진입) |
| C8 | **이미 등록된 환자 재등록 차단** — "이미 등록된 환자예요" 표시하고 요청 안 보냄 |
| C9 | **회원 동의 화면 재설계**(§2.7) |
| **C10** | **모든 화면 = 관리자 대시보드(/admin) UI/UX·디자인 토큰 그대로** (Stat/Tool 카드·필터폼·기간칩·테이블·`--ink/--tt-blue/--line`) |
| **C11** | **매칭 실패 복구** — 생년월일 등 회원 오입력으로 매칭 실패 시, 회원이 본인 계정(설정/온보딩)에서 생년월일 수정 → 병원이 재등록 요청하면 연결됨 |
| **C12** | **생년월일 입력 = 드롭다운 + 직접 타이핑 자유**(790126·1979-01-26 등 유연 파싱) |
| **C13** | 모든 기록 추후 수정 가능 · **수정·삭제는 회원 알림 미발송**(작성만 발송) · 시술기록 대장 기본 = **방문일 최신순** |
| **C14** | 명칭 = **"환자 관리" · "시술기록 관리"**(‘DB’ 표기 안 씀) |

## 1. 정보구조 (라우트 트리)

```
/clinic                                     대시보드 — 현황 4카드 + 운영 프로그램 카드
│
├─ 환자 관리
│   /clinic/patients                        환자 목록(검색·정렬·필터 테이블)
│   /clinic/patients/new                    환자 등록
│   /clinic/patients/[linkId]               환자 상세 = 프로필 + 그 환자 시술기록 타임라인
│
└─ 시술기록 관리
    /clinic/visits                          지점 전체 시술기록(목록 뷰 + 캘린더 뷰)
    /clinic/visits/new                      시술노트 작성(환자 선택 → DiaryForm)
    /clinic/patients/[linkId]/visits/[id]/edit   기록 1건 열람·수정·삭제
```
- 공용 재사용: `requireClinicPage`·`ClinicShell`·`layout`(noindex)·`DiaryForm(mode='clinic')`·0345 RPC 패턴.
- 서버 렌더 + `searchParams`(q·status·sort·dir·period·view·page) 필터 SSOT.

### 2.2 환자 등록 `/clinic/patients/new`
- 폼: 회원 아이디(입력칸만 — **안내문구 없음**) · 이름 · 생년월일(**드롭다운+직접입력, C12**) · 등록번호(선택).
- 안내 카피(원장 수정): **"입력한 회원에게 동의 요청 알림이 발송돼요."**
- **중복 차단(C8)**: 제출 시 이미 pending/active 면 "이미 등록된 환자예요 · {이름}({상태}) →"(그 환자로 이동) 표시하고 요청 안 보냄(서버 409 를 완료로 처리).

### 2.3 환자 목록 `/clinic/patients` — 검색·정렬 테이블

### 2.5 시술기록 관리 `/clinic/visits` — 목록 + 캘린더(C7) · 하루/한주/한달 조회
**2뷰 토글**(`?view=list|calendar`). 기본 **방문일 최신순(C13)**.
- **기간 선택 3방식**(모두 내부적으로 날짜 범위 `from~to` 로 통일, URL `?from=&to=`):
  - ⓐ **빠른 버튼** 4종 — 오늘 / 한 주 / 한 달 / 전체(전체=범위 없음).
  - ⓑ **년·월 바로가기 드롭다운** — `[YYYY년▾][M월▾]` 선택 시 그 달로 점프(from=1일·to=말일). 캘린더 ◀▶ 로 한 달씩 넘길 필요 없이 과거 특정 달 즉시 이동(클릭 최소화).
  - ⓒ **직접 범위 지정** — 시작일~종료일(캘린더 클릭·드래그 또는 date input 2개).
- **ⓐ 목록 뷰**(admin 테이블): 방문일 · 환자(클릭→그 환자 기록 C6) · 시술요약 · 원장 · 금액 · 다음예약. 선택한 기간의 기록을 방문일순으로. 필터=원장별·검색(환자·시술).
- **ⓑ 캘린더 뷰**: 월간 캘린더, 날짜 셀에 그날 기록 수. **날짜 클릭=그 하루**, **주 선택=그 한 주**, **월 이동=그 한 달** 로 목록이 바뀜(같은 범위 방식). 행 클릭→편집/환자 진입. 기존 캘린더 컴포넌트 재사용 검토.
```
시술기록 관리                    [목록][캘린더]     이번 달 48건
[🔍 환자·시술] [오늘][한주][한달][전체]  [2026년▾][7월▾]  범위[07.03~07.09] [원장▾]
┌────────────┬──────┬────────────┬──────┬────┬────────────┐
│방문일 ▾    │환자  │시술         │원장  │금액│다음예약    │
│2026.07.06  │김서연│울쎄라·리쥬란│정한미│85만│2026.08.10 │
└────────────┴──────┴────────────┴──────┴────┴────────────┘
캘린더 뷰: 날짜 클릭→그날 / 주 선택→그주 / 월 이동→그달
```

### 2.6 시술기록 편집 `/clinic/patients/[linkId]/visits/[id]/edit`
**DiaryForm(mode='clinic') 재사용** + 초기값 주입 + PATCH. **날짜(방문일·다음예약) 최상단**(원장 핵심 요구, C13). 시술 추가/삭제·원장·메모·금액 수정. 하단 "이 기록 삭제"(확인 모달, 후기 있으면 차단 C5). 저장·삭제 후 `router.push` 환자 상세 복귀. **수정·삭제 알림 미발송(C13)**.
```
← 환자 상세    시술노트 수정 — 김서연님
 방문일     [ 2026-07-06 ]  (달력)      ← 최상단
 다음 예약일 [ 2026-08-10 ]
 담당 원장   [ 정한미 원장 ▾ ]
 시술 [울쎄라 ✕][리쥬란 ✕] [+ 추가]   메모 […]   총액 [850000]
 [ 수정 저장 ]                         [ 이 기록 삭제 ]
```

## 4. DB 계층 (마이그 0350 · 0351) + 공용 컴포넌트

| RPC | 시그니처(요약) | 보안 경계 |
|---|---|---|
| `get_clinic_patient_visits` | (clinic_profile, link_id) → 그 환자 기록+procedures | `source='clinic' AND clinic_id=자기 AND profile_id=연결회원` **3중** |
| `get_clinic_visits`★ | (clinic_profile, search, doctor_id?, **p_from date, p_to date**, sort, dir, limit, offset) → 지점 전체 | `source='clinic' AND clinic_id=자기` · 범위=하루/한주/한달/전체(from~to null=전체) |
| `clinic_update_visit` | (clinic_profile, diary_id, visited_on, procedures, doctor_id?, …) 전체교체 | 3중 + **revoked 차단(C2)** + 재직원장만 · 불변컬럼 미수신 |
| `clinic_delete_visit` | (clinic_profile, diary_id) | 3중 + **후기 있으면 차단(C5)** · 미발송 |
| `get_clinic_patients` **v2** | (+status_filter·sort_by·sort_dir·limit·offset) → +`last_visit_on`·`visit_count`·나이 파생 | 하위호환(DEFAULT) |

- **검색 강화(원장)**: `get_clinic_patients` 검색 = 이름·등록번호·아이디 부분 ILIKE + **생년월일은 클라가 파싱해 `date` 로 전달→동등비교**(§4.2-6, 인덱스 활용). 클라는 `790126`·`19790126`·`1979-01-26` 을 `1979-01-26` 로 정규화(2자리 연도 피벗). `to_char ILIKE` 는 seq scan·2자리연도 모호로 폐기.
- **인덱스**: `diaries(clinic_id, visited_on DESC) WHERE source='clinic'` · `clinic_member_links(clinic_id, patient_birthdate)`(생일 동등검색).
- **불변 컬럼 방어**: update RPC 는 `source/clinic_id/profile_id/created_by/linked_consent_at/created_at` 를 파라미터로 안 받음.
- 적용: 0350(기록 RPC 3종+인덱스) → 0351(patients v2). 한글 RAISE → UTF-8 경로 필수.

### 4.1 공용 컴포넌트 신설·확장
- **`BirthdateSelect` 확장(C12)**: 드롭다운 + **직접 타이핑**(텍스트 입력 — `790126`·`19790126`·`1979-01-26`·`1979.01.26` 유연 파싱 → `YYYY-MM-DD` 정규화, 2자리 연도는 현재연도 피벗). 온보딩 회귀 없이 clinic 등록에 적용(기존 사용처 하위호환).
- **admin `Stat`·`Tool` 카드 추출**(C10): `AdminView` 로컬 컴포넌트를 공용 모듈로 → admin·clinic 공유.

## 4.2 DB 검토 반영 — 3인 독립 진단 종합 (정정)

세 검토관이 실제 스키마·운영 DB로 교차 검증. **[치명] 4건은 서로 독립 확인** → 착수 전 설계 반영.

**[치명] (S1 착수 전 확정):**
1. **`get_clinic_patients` v2 = `DROP` 후 재생성** — RETURNS TABLE 에 컬럼 추가(last_visit_on·visit_count·age)는 `CREATE OR REPLACE` 불가(`cannot change return type`). 0351 맨 앞 `DROP FUNCTION IF EXISTS public.get_clinic_patients(uuid, text);` → 재생성. **동시 갱신 필수**: `ClinicPatientItem`(TS)·`/api/clinic/patients` 라우트(sort·dir·status·limit·offset 수신→RPC 전달)·소비 3파일 한 배치.
2. **회원 편집 경로(C4) 감사컬럼 보존** — `update_visit`(0297)이 `clinic_id=p_clinic_id` 로 덮어씀(실측 0297:393). source='clinic' 노트를 회원이 편집하면 병원 귀속(clinic_id) 훼손 → 병원이 자기 노트를 잃음. **정정**: `update_visit` 을 source='clinic' 이면 `clinic_id/source/created_by_clinic_profile_id/linked_consent_at` 보존하도록 패치. (현재 회원 diary 편집 UI 부재라 latent — C4 편집 UI 신설과 **동시** 처리, S3.)
3. **`clinic_update_visit`·`clinic_delete_visit` = 신규 필수** — 기존 `delete_visit`(0297)은 병원이 못 씀(clinic_id 검증 없어 타 지점 횡단 위험). 0350 신규 구현 검증 순서: ① 병원 명함 `auth.uid()` ② diaries `source='clinic' AND clinic_id=자기 AND profile_id=연결회원`(**3중 — profile_id 누락이 최빈 실수**) ③ `clinic_member_links` `FOR UPDATE` + `status='active'` 만(revoked 차단 C2) ④ delete 는 후기 있으면 차단(C5) + `delete_visit` 후기 standalone 전환 로직 재사용.
4. **캘린더 집계 전용 RPC** — `get_clinic_calendar_summary(clinic_profile, p_year, p_month) → (visit_date date, visit_count bigint)` 신설(0350, GROUP BY visited_on). 월 전체 행 전송 회피. 날짜 클릭 시 `get_clinic_visits(period='day')`.

**[중요] (구현 반영):**
5. **동적 ORDER BY 인젝션 차단** — `p_sort_by`·`p_sort_dir` 은 `CASE` 화이트리스트로만 ORDER BY 구성 + `p_sort_dir IN ('asc','desc')` 검증. `patient_name` 은 `COLLATE "ko-x-icu"`(운영 DB collation 존재 사전 확인).
6. **생일 검색 = 클라 파싱 우선** — 완전한 생일(6·8자리·YYYY-MM-DD)이면 클라가 DATE 로 파싱→RPC 에 `p_birthdate` 로 전달→`patient_birthdate = 파싱값` 동등비교(인덱스 활용). 부분·이름·등록번호는 ILIKE. `to_char ILIKE`(seq scan·2자리연도 모호) 회피.
7. **인덱스** — `diaries(clinic_id, visited_on DESC) WHERE source='clinic'`(신규·대장 정렬) + `clinic_member_links(clinic_id, patient_birthdate)`(생일 검색). 기존 `diaries_clinic_idx`(clinic_id 부분)와 목적 분리 주석. 이름 `ILIKE '%..%'` 양쪽 와일드카드는 B-tree 미활용 → 현 규모 seq scan 허용, `pg_trgm` GIN 은 ROADMAP.
8. **후기 달린 clinic 노트 = 병원 수정도 차단** — `diary_procedures` 전체 교체가 `procedure_reviews.diary_procedure_id`(ON DELETE SET NULL, 0292)를 끊어 회원 후기의 시술연결 소실. C5(삭제 차단)와 대칭으로 **수정도 `visit_has_linked_reviews` 차단**.
9. **집계 clinic_id 필터 필수** — `last_visit_on`/`visit_count` 서브쿼리에 `AND clinic_id=자기`(타 병원 노트 혼입 방지). 성별=`patient_skin_profile.gender`, pending 은 NULL.
10. **C11 재연결 status 분기** — rejected/revoked 는 재요청 가능(부분 UNIQUE 는 pending/active 만). 중복 안내(C8)는 status 별 분기: pending/active=차단·"이미 등록", rejected/revoked=재요청 가능 안내. `/clinic/patients/new` 409 응답에 status 포함.
11. **대장 기간 = `visited_on`(시술일) 범위 `p_from`~`p_to`** — 하루/한주/한달/전체를 이 범위로 표현(고정 enum 대신 유연). 캘린더 하루=from=to=그날, 한주=주 시작~끝, 한달=월 1일~말일. ⚠ 대시보드(0349) "오늘/이번 달 기록"은 `created_at`(작성일) 기준 — **의미 다름**(주석 명시).
12. **관례** — 신규 함수 `REVOKE ALL … FROM PUBLIC; GRANT EXECUTE … TO authenticated;` + `search_path` 고정 + 한글 RAISE 는 UTF-8 경로(scratchpad/db.mjs)·U+FFFD 0 재스캔. offset 페이지네이션(1차) + keyset 커서 파라미터(`p_after_visited_on`·`p_after_id` DEFAULT NULL) 예약.
