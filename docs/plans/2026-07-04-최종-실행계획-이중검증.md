2. 스냅샷(generated.json) 재생성이 포함된 커밋(R1-3, R4-3)은 다른 변경과 절대 혼합 금지.
3. dual-SSOT 페어(CLAUDE.md §5)를 건드리는 커밋은 페어 표를 먼저 확인, 한 커밋 동시 갱신.
4. 비-ASCII 마이그는 UTF-8 경로(node fetch) + U+FFFD 재스캔(CLAUDE.md §8).
5. 각 커밋: build + tsc + code-reviewer(치명 시 재검수) 통과 후 push. R4-2(보안 로직 인접)는 검수 중점 지정.
6. 라운드 종료마다 CHANGELOG.md + 해당 도메인 문서 갱신, 이 문서의 체크 상태 갱신.
