## 3. 남은 작업 (Phase 2~6) — 전부 Medium/Low, 사고 아님

### Phase 2 — 정합성·데이터 (지뢰성 우선)

- **M-1 `cards.category` 기본값 오류** — DB 컬럼 DEFAULT 가 폐기된 `'diary'` 라 현행 CHECK(qa/doodle/review/review_summary) 자기위반. 지금은 모든 INSERT 가 category 명시라 잠복이나, 생략 경로 생기면 500. → `ALTER TABLE cards ALTER COLUMN category SET DEFAULT 'doodle'`(또는 DROP DEFAULT) 마이그.
- **M-7 앵커 카드 404 (실제 눈에 보임)** — 카드 공개 URL 규칙이 SSOT(`lib/card-url.ts::getQaUrl`, SQL `card_public_url`) 외에 `app/cards/[id]/page.tsx`·`admin/stats/[kind]/StatsListClient.tsx` 2곳에 복제됐는데 **둘 다 review_summary 분기 누락** → 관리자 통계/카드ID 진입 시 리포트 앵커(발행 65건)가 `/reports` 대신 404. → 두 사본에 review_summary→`/reports/{post_slug}` 분기 추가(SSOT 재사용).
- **M-2 오타교정 역방향 7행** — `tag_normalization` 에 현행 대표어(ko)를 입력키(canonical)로 쓰는 레거시 7행(고압산소치료·리쥬란HB·리프팅시술·마리오네트주름·보톡스내성·보톡스주기·콜라겐자극). 리쥬란HB 는 0318 자가모순. `normalizeTag` 가 정상 태그를 자기 alias 로 되돌림. → 7행 방향 정정/삭제 마이그 + 생성 스냅샷 재생성.
- **M-3 `/api/articles` type↔category 미강제** — 둘을 각각만 검증, 짝(qa↔qa/post↔doodle) 미검사. 현재 mismatch 0이나 방어선 없음. → 라우트에서 교차 검증 또는 category→type 파생.
- **환경변수·문서 stale** — 코드 참조 env 8종(YOUTUBE_OAUTH_*/PYTHON_*/NCBI_API_KEY 등)이 `.env.local.example`·`DEPLOYMENT.md §10` 양쪽 부재 + `NEXT_PUBLIC_NAVER_MAP_CLIENT_ID` 죽은 항목. **README** Production URL 이 구 `pbtt.kr`. ARCHITECTURE/PRD 에 없는 라우트(settings/notifications·api/dev-sql)·`primary` 쿠키·user_id 표기·PII 컬럼 수 stale. → 문서 동기화(§5 페어 규칙).
- (완료) reserved_handles 확장 — 이번 세션 처리됨.
