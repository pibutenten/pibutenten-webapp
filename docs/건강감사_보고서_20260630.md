## Part 2. 강점 — 기준선 (무엇을 깨면 안 되는가)

수정 작업 시 아래 모범 패턴을 **훼손하지 않도록** 기준선으로 명시합니다.

### 보안
- **SSRF 방어**(`ssrf-guard.ts`): 단순 hostname 매칭이 아니라 **DNS 해석 후 모든 결과 IP 검사** + IPv4/IPv6 사설·메타데이터 대역 차단 + **redirect 매 hop 재검증**(`redirect:'manual'`) + maxBytes 스트림 abort. og-extract/preview-link/place-search 가 이를 경유.
- **업로드**(`api/upload`): magic byte 검증 + sharp EXIF 제거·재인코딩 + 8MB·rate limit(20/분) + `crypto.randomUUID` 파일명. 헬스케어 환부사진 GPS 유출 차단 의식.
- **CSRF**(`middleware.ts`): unsafe method 에 Origin/Referer 화이트리스트 검증(환경변수화, 서브도메인 공격 차단).
- **RLS/RPC**: SECURITY DEFINER 전수 `SET search_path` 고정, admin RPC `is_admin()` 가드, service_role 키 server-only 격리, anon PII 컬럼 화이트리스트(0123).

### 안정성
- **하이드레이션 방어 패턴**: `RelativeTime`·`SessionContext`·`useWeather` 가 Date/random/localStorage 를 "마운트 후 setState + suppressHydrationWarning" 으로 처리.
- **fail-safe 미들웨어**: `auth.getUser()`/profiles 조회 예외 시 가드 스킵(무한 redirect 방지), `site_visits` 는 `event.waitUntil` 비블로킹 best-effort.
- **방어적 storage**: recent-search·weather 헬퍼가 JSON.parse 후 런타임 형태 검증.

### 성능 (핵심 경로)
- 홈(`/`)·`/today`·`/[handle]`·`/doctors/[slug]`·알림·댓글이 `unstable_cache`/React `cache`/`Promise.all` 로 최적화. 홈 인기태그 `feed_cards_scored` 는 `unstable_cache(revalidate 300)` 로 감쌈. FeedView 댓글·뷰어 N+1 배치 제거.

### 정합성(SSOT)
- `categories.ts`(9색 UI 칩)↔`post-category.ts`(글 카테고리)를 "혼동 주의" 주석으로 명확 분리. `card-url.ts::getQaUrl`↔`card_public_url` SQL dual-SSOT 로 화면 링크와 트리거 저장 URL 을 일치.

## Part 3. [신규] 회원가입 튕김 버그 정밀 진단 (제보 반영)

> 제보: *"앱에서 회원가입창이 자꾸 튕겨서 다른 페이지로 간다."* → **재현·원인 확정.** 직접 확인한 실재 버그입니다. (기존 `수정제안서_20260630.md` #3 과 동일 뿌리이나, 이번엔 **앱 특화 2차 원인**까지 규명.)

### Step 1 — 전체 맥락 (auth 흐름 caller 추적)

이 제품의 가입 = **소셜 로그인**입니다. `/signup` 은 "가입 폼"이 아니라 **OAuth 성공 후 약관 동의 게이트**로, **인증된 세션이 있어야만** 동작합니다.

- 정상 경로: `/login` 소셜 버튼 → `/auth/callback`(`exchangeCodeForSession`) → 세션 발급 → `auth/callback/route.ts:337` `if (!profile || !terms_agreed_at) redirect('/signup')` → `SignupForm`(약관) → `/onboarding`.
- 즉 `/signup` 은 **항상 로그인된 상태에서만** 도달하도록 설계됨.

### Step 2 — 현재 문제의 구체적 위치 (두 겹)

**① 직접 원인 — 비로그인 사용자가 밟는 튕김 루프**
- `src/app/login/LoginForm.tsx:153-154`: "처음이신가요? **회원가입**" 링크가 `href="/signup"` 을 가리킴.
- `src/app/signup/page.tsx:28-30`: `if (!user) redirect('/login...')`.
- **결과**: 비로그인 방문자가 "회원가입"을 누르면 → `/signup` → (user 없음) → `/login` 으로 즉시 되돌아옴. 화면엔 "눌렀는데 깜빡이고 제자리/다른 데로 감". **이것이 제보된 튕김입니다.**

**② [정정] 앱 소셜 OAuth 는 실제로 구현돼 있음 — 남은 건 외부 설정 확인**
> ⚠️ 초판에서 "인앱 OAuth 미완성"이라 적었으나, 코드 전수 확인 결과 **오판이었습니다**. `capacitor.config.ts` 의 낡은 주석 한 곳만 보고 내린 결론이었고, 그 주석이 stale 입니다. 정정합니다.
- `SocialLoginButtons.tsx:83-101`: 네이티브 감지 시 `@capacitor/browser` 로 OAuth 를 **시스템 브라우저**에 띄우고(`signInWithOAuth({ redirectTo: NATIVE_OAUTH_CALLBACK, skipBrowserRedirect:true })`), 복귀는 커스텀 스킴 `kr.pibutenten.app://auth/callback`.
